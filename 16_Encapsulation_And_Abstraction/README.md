# 16. Encapsulation and Abstraction

- **Encapsulation**: bundling data and the methods that operate on it,
  while restricting direct access to internal details (data hiding).
- **Abstraction**: exposing only essential features, hiding implementation
  complexity behind a simple interface.

## Files

| File | Description |
|------|--------------|
| `01_encapsulation_getters_setters.cpp` | Private data accessed via public getter/setter methods |
| `02_abstraction_via_interface.cpp` | Hiding implementation behind an abstract interface |

## Key Takeaways
- Encapsulation protects an object's internal state from invalid external modification.
- Getters/setters can add validation logic that direct field access cannot.
- Abstraction lets client code depend on WHAT an object does, not HOW it does it.
